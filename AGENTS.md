# Agent Protocol

## Commit & Testing Procedures

1. Whenever you make a change, make sure to git commit your work before responding back to the user.
2. Use industry standards while writing the commit message, use terms like feat, fix, chose etc. The git message should be detailed and descriptive.
3. For every bug and feature we add or report, create a dedicated test case to prevent regressions.
4. When deviating from a plan, please take permission from me.

## Environment & Communication

1. Use `uv` as the default package manager, falling back to `python3` or `python` if necessary.
2. If a requested change seems suboptimal, point it out and provide a rationale.
